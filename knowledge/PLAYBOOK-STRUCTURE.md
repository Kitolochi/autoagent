# Social Media Editing Playbook - Structure

This playbook is Claude's operational manual for producing professional video edits. It defines the systematic knowledge needed to make consistent, high-quality editorial decisions during automated video production.

---

## 1. Video Type Taxonomies

Each video type has a distinct purpose, audience expectation, pacing profile, and technique palette.

### 1.1 Primary Video Types

#### Demo (Product Demonstration)
**Purpose**: Show the product working in real-time  
**Core Pattern**: Problem → Product → Proof  
**Pacing**: Medium (4-8s per feature)  
**Essential Techniques**:
- 3D Tilt Entry (product reveal)
- UI Reveal (layered panel build)
- Live Typing (real-time interaction)
- Feature Highlight (callout annotations)
- Morph Transition (between states)
- Negative Space (clarity)

**Music Profile**: Clean, confident, minimal percussion  
**Text Style**: Sans-serif, medium weight, labels + annotations  
**Color Grading**: High contrast, saturated product UI  
**Template Checklist**:
- [ ] Hook: Problem statement (0-3s)
- [ ] Product entrance with 3D tilt (3-8s)
- [ ] 2-3 feature demonstrations with callouts (8-35s)
- [ ] Visual proof element (full-circle payoff at 60-80%)
- [ ] CTA on light background (final 3-5s)

#### Hype (Launch/Announcement)
**Purpose**: Generate excitement and energy around a release  
**Core Pattern**: Manifesto → Impact → Call to Action  
**Pacing**: Fast (1-3s per scene, beat-synced)  
**Essential Techniques**:
- Beat-Synced Cut (every musical beat)
- Kinetic Type (text as primary visual)
- Cascading Windows (show breadth)
- Zoom Punch (impact moments)
- Data Bloom (proof numbers)
- Light/Dark Alternation (rhythm)

**Music Profile**: High energy, clear beats, 120-140 BPM  
**Text Style**: Bold display type, 80-120px, animated  
**Color Grading**: High saturation, crushed blacks  
**Template Checklist**:
- [ ] Dark opening with manifesto text (0-3s)
- [ ] Beat-synced product montage (3-20s)
- [ ] Data bloom moment (proof of impact)
- [ ] Cascading windows (breadth signal)
- [ ] Dark closing with CTA (final 3-5s)

#### Tutorial (How-To/Educational)
**Purpose**: Teach a specific skill or workflow  
**Core Pattern**: Context → Steps → Outcome  
**Pacing**: Slow (8-15s per step)  
**Essential Techniques**:
- Problem-Based Hook
- Feature Highlight (step annotations)
- Live Typing (input demonstration)
- Morph Transition (step progression)
- Negative Space (clarity)
- Split View (before/after)

**Music Profile**: Calm, minimal, non-distracting  
**Text Style**: Clear sans-serif, readable at mobile size  
**Color Grading**: Neutral, prioritize UI legibility  
**Template Checklist**:
- [ ] Hook: Problem statement (0-5s)
- [ ] Step 1 demonstration with annotation (5-15s)
- [ ] Step 2 demonstration with annotation (15-25s)
- [ ] Step 3 demonstration with annotation (25-35s)
- [ ] Result with before/after comparison (35-45s)
- [ ] CTA (final 3-5s)

#### Compare (Before/After, Competitive)
**Purpose**: Show improvement or differentiation  
**Core Pattern**: Old State → New State → Delta  
**Pacing**: Medium (5-10s per comparison)  
**Essential Techniques**:
- Split View (side-by-side)
- Morph Transition (state change)
- Full-Circle Visual Payoff (same frame, different data)
- Problem-Based Hook
- Data Bloom (quantified improvement)

**Music Profile**: Dramatic build, clear transition moment  
**Text Style**: Labels for states ("Before" / "After")  
**Color Grading**: Desaturate old, saturate new  
**Template Checklist**:
- [ ] Hook: Pain point in old state (0-5s)
- [ ] Split view setup (5-8s)
- [ ] Side-by-side comparison with labels (8-20s)
- [ ] Morph to new state with data bloom (20-30s)
- [ ] Full-circle payoff (same UI, better results) (30-40s)
- [ ] CTA (final 3-5s)

#### Testimonial (Social Proof)
**Purpose**: Build credibility through user stories  
**Core Pattern**: Person → Problem → Solution → Result  
**Pacing**: Medium-slow (6-12s per testimonial)  
**Essential Techniques**:
- Data Bloom (results/metrics)
- Decorative Symmetry (centered quote)
- Negative Space (focus on words)
- Light/Dark Alternation (mood shifts)

**Music Profile**: Warm, human, gentle build  
**Text Style**: Serif or elegant sans for quotes  
**Color Grading**: Warm tones, softer contrast  
**Template Checklist**:
- [ ] Hook: Relatable problem (0-5s)
- [ ] Quote card with person attribution (5-15s)
- [ ] Product demonstration (15-25s)
- [ ] Data bloom with results (25-35s)
- [ ] Additional testimonials (if multiple) (35-45s)
- [ ] CTA (final 3-5s)

#### Brand (Identity/Values)
**Purpose**: Communicate who you are, not what you sell  
**Core Pattern**: Philosophy → Atmosphere → Invitation  
**Pacing**: Slow (8-15s per concept)  
**Essential Techniques**:
- Kinetic Type (value statements)
- Beat-Synced Cut (philosophical rhythm)
- Light/Dark Alternation (mood)
- Atmospheric elements (blur, parallax, particles)
- Manifesto-Demo-Manifesto (A-B-A structure)

**Music Profile**: Cinematic, emotional, orchestral or ambient  
**Text Style**: Display serif or refined sans  
**Color Grading**: Moody, low contrast, branded colors  
**Template Checklist**:
- [ ] Dark opening with brand statement (0-5s)
- [ ] Atmospheric visuals with philosophy text (5-20s)
- [ ] Optional product glimpse (20-30s)
- [ ] Return to philosophy on light bg (30-40s)
- [ ] Brand logo + tagline (final 3-5s)

#### Explainer (Concept/Process)
**Purpose**: Make complex ideas understandable  
**Core Pattern**: Question → Breakdown → Synthesis  
**Pacing**: Medium (6-10s per concept)  
**Essential Techniques**:
- Problem-Based Hook
- Negative Space (clarity)
- Morph Transition (concept evolution)
- Feature Highlight (diagram annotations)
- Data Bloom (key statistics)

**Music Profile**: Thoughtful, building, optimistic  
**Text Style**: Clean sans-serif, hierarchical sizing  
**Color Grading**: Clear, saturated primaries for diagrams  
**Template Checklist**:
- [ ] Hook: Question or curiosity gap (0-3s)
- [ ] Concept 1 with visual metaphor (3-12s)
- [ ] Concept 2 with visual metaphor (12-21s)
- [ ] Concept 3 with visual metaphor (21-30s)
- [ ] Synthesis: how it all connects (30-40s)
- [ ] CTA (final 3-5s)

#### Recap (Monthly/Quarterly Summary)
**Purpose**: Communicate momentum and progress  
**Core Pattern**: Achievements → Highlights → Forward Look  
**Pacing**: Fast (2-5s per achievement)  
**Essential Techniques**:
- Beat-Synced Cut (energetic pacing)
- Cascading Windows (breadth of work)
- Data Bloom (metrics/milestones)
- Kinetic Type (achievement headlines)
- Light/Dark Alternation (rhythm)

**Music Profile**: Upbeat, celebratory, 120-140 BPM  
**Text Style**: Bold sans-serif, consistent headlines  
**Color Grading**: Vibrant, high energy  
**Template Checklist**:
- [ ] Hook: Big number or achievement (0-3s)
- [ ] Achievement montage with beat-synced cuts (3-25s)
- [ ] Cascading windows of shipped work (25-35s)
- [ ] Forward-looking statement (35-40s)
- [ ] CTA or next milestone (final 3-5s)

### 1.2 Hybrid Types

Some videos blend multiple types:

- **Demo + Tutorial**: Show it working, then teach how to use it
- **Hype + Testimonial**: Launch energy with social proof
- **Explainer + Demo**: Explain the concept, demonstrate the tool
- **Brand + Demo**: Philosophy opener, product demo middle, philosophy closer

**Rule**: When combining types, use the primary type's pacing for 70%+ of runtime. Secondary type appears in specific sections (opener/closer, middle proof moment).

## 2. Element Libraries

Reusable components organized by function.

### 2.1 Transition Catalog

Transitions manage viewer attention during scene changes. Each transition type signals different editorial intent.

#### Beat-Synced Cut
**When to Use**: Hype videos, recap videos, high-energy moments  
**Duration**: 0ms (instant cut on musical beat)  
**Implementation**: Align cut to transient in audio waveform  
**Frequency**: 1 cut per beat during peak energy sections  
**Value**: 5 for Hype/Brand/Recap, 2 for Tutorial/Explainer  
**Anti-Pattern**: Using in slow, educational content (creates chaos)

#### Morph Transition
**When to Use**: Demo, compare, explainer (logical progression)  
**Duration**: 400-800ms  
**Implementation**: Element from Scene A transforms into element in Scene B  
**Frequency**: 2-4 per video  
**Value**: 5 for Demo/Compare, 4 for Explainer  
**Anti-Pattern**: Morphing unrelated elements (breaks logic)

#### Geometric Wipe
**When to Use**: Universal safe default  
**Duration**: 300-500ms  
**Implementation**: Shape-based reveal (circle expand, diagonal, grid)  
**Frequency**: Unlimited  
**Value**: 3-4 across all types  
**Variants**:
- Circle expand: organic, friendly
- Diagonal slash: dynamic, directional
- Grid reveal: technical, precise

#### Zoom Punch
**When to Use**: Hype videos, impact moments  
**Duration**: 200ms  
**Implementation**: Scale 1.0→1.3 + slight blur + cut  
**Frequency**: Max 4 per video (loses impact if overused)  
**Value**: 5 for Hype, 1 for Tutorial  
**Anti-Pattern**: Using more than once per 10s (feels frantic)

#### Fade (Cross-Dissolve)
**When to Use**: Testimonials, brand videos, time passage  
**Duration**: 600-1200ms  
**Implementation**: Opacity blend between scenes  
**Frequency**: Unlimited in slow-paced content  
**Value**: 4 for Brand/Testimonial, 2 for Hype  
**Anti-Pattern**: Using in fast-paced content (feels sluggish)

#### No Transition (Hard Cut)
**When to Use**: Maintain rhythm, preserve energy  
**Duration**: 0ms  
**Implementation**: Direct cut without effect  
**Frequency**: Unlimited  
**Value**: Universal  
**When**: Beats are handled by music, or scene change is subtle

### 2.2 Music Selection Guide

Music dictates pacing, energy, and emotional tone. Match music to video type and editorial intent.

#### Mood Mapping

| Video Type | Primary Mood | BPM Range | Instrumentation | Energy Arc |
|---|---|---|---|---|
| Demo | Confident, Clean | 100-120 | Minimal percussion, synth bass | Steady |
| Hype | Excited, Bold | 120-140 | Heavy beats, synth leads | Build to peak |
| Tutorial | Calm, Focused | 80-100 | Acoustic, soft electronic | Flat (non-distracting) |
| Compare | Dramatic, Revealing | 90-110 | String swells, percussion hits | Build at transition |
| Testimonial | Warm, Human | 70-90 | Piano, guitar, strings | Gentle rise |
| Brand | Cinematic, Emotional | 60-100 | Orchestral or ambient | Slow build |
| Explainer | Thoughtful, Optimistic | 95-115 | Marimba, bells, light synth | Progressive |
| Recap | Celebratory, Upbeat | 120-140 | Full band, synth, brass | High throughout |

#### Beat Alignment Rules

- **Hype/Recap**: Align major cuts to beat transients (kick, snare)
- **Demo/Tutorial**: Use music as background, don't force beat-syncing
- **Brand**: Align philosophy text cards to musical phrases (4-8 bar sections)
- **Compare**: Time the "reveal" moment to musical climax or drop

#### Music Structure Alignment

Most stock music follows 8-bar phrase structure:
- **0-8s**: Intro (sparse)
- **8-24s**: Build (add layers)
- **24-40s**: Peak (full instrumentation)
- **40-48s**: Outro (reduce)

**Editorial Strategy**:
- Place hook in intro (0-8s)
- Core content in build/peak (8-40s)
- CTA in outro (40-48s)

### 2.3 Text & Graphics Patterns

Typography is functional. Text must be readable, purposeful, and timed to content.

#### Lower Thirds (Name/Title Cards)
**When to Use**: Testimonials, interviews, speaker identification  
**Placement**: Bottom 20% of frame, left-aligned or centered  
**Duration**: 3-5s on screen  
**Style**: Sans-serif, medium weight, 24-36px  
**Animation**: Slide in from left (300ms), hold, slide out to left (300ms)  
**Implementation**:
```
Name: 48px Bold
Title: 32px Regular
Org: 28px Light
```

#### Call-to-Action (CTA)
**When to Use**: Final 3-5s of every video  
**Placement**: Center frame or bottom third  
**Duration**: 3-5s  
**Style**: High contrast, button-style background  
**Variants**:
- "Learn More" → product demos
- "Get Started" → tutorials
- "Join Us" → brand videos
- "Try Free" → competitive comparisons

**Animation**: Scale in with slight bounce (easeOutBack)

#### Feature Labels
**When to Use**: Demos, tutorials (annotating UI)  
**Placement**: Adjacent to UI element being highlighted  
**Duration**: 2-4s per label  
**Style**: Small sans-serif (18-24px), white text on semi-transparent dark bg  
**Animation**: Fade in (200ms), hold, fade out (200ms)

#### Kinetic Typography (Animated Words)
**When to Use**: Hype, brand, recap (text as primary visual)  
**Placement**: Center frame, full-screen treatment  
**Duration**: 1-3s per word/phrase  
**Style**: Bold display type, 80-120px  
**Animation**: Scale + rotation + position per word  
**Implementation Pattern**:
- Word 1: Scale from 0 to 1 with rotation (300ms)
- Hold (600ms)
- Word 2: Overlaps at 700ms mark
- Word 1 exits while Word 2 enters

#### Data/Stats Display
**When to Use**: Hype, testimonial, recap (proof moments)  
**Placement**: Center frame  
**Duration**: 3-5s  
**Style**: Large bold number (100-150px) + context line (36px)  
**Animation**: Counter from 0 to target value (easeOutQuart, 1200ms)  
**Example**:
```
"$10M" (number, 120px bold)
"Revenue in Year 1" (context, 36px regular)
```

### 2.4 Color Grading Presets

Color creates mood and guides attention. Grading should match video type and brand.

#### High Contrast (Demo/Hype)
**Purpose**: Maximize clarity and energy  
**Settings**:
- Lift shadows: -10
- Gamma: 1.1
- Gain highlights: +15
- Saturation: +20
**Use When**: Product UI needs to pop, high-energy content

#### Neutral/Natural (Tutorial/Explainer)
**Purpose**: Prioritize legibility, avoid distraction  
**Settings**:
- Minimal adjustment
- Slight saturation reduction (-5)
- Ensure UI elements remain readable
**Use When**: Educational content, screen recordings

#### Warm/Human (Testimonial)
**Purpose**: Create emotional connection  
**Settings**:
- Lift shadows: +5
- Add warm tint (orange/yellow)
- Saturation: +10 in midtones
- Softer contrast (reduce blacks)
**Use When**: People-focused content, personal stories

#### Moody/Cinematic (Brand)
**Purpose**: Establish atmosphere and sophistication  
**Settings**:
- Crushed blacks (lift shadows: -20)
- Reduced saturation (-10 to -20)
- Color tint (teal/orange, or brand colors)
- Lower gamma (0.9)
**Use When**: Brand storytelling, philosophical content

#### Desaturate/Saturate (Compare)
**Purpose**: Visual signal of improvement  
**Settings**:
- "Before" state: -30 saturation, slight blue tint
- "After" state: +20 saturation, slight warm tint
**Use When**: Before/after comparisons, competitive positioning

## 3. Checklists

Systematic checks at each production stage prevent common failures.

### 3.1 Pre-Edit Checklist

Run before starting the edit. Ensures assets and plan are ready.

#### Asset Verification
- [ ] All footage files accessible and playable
- [ ] Audio files present and correct format
- [ ] Graphics/logos in high resolution (min 1080p)
- [ ] Music track matches desired BPM and mood
- [ ] Script/voiceover (if applicable) matches video type
- [ ] Brand colors and fonts identified

#### Shot Selection
- [ ] Best takes selected for each scene
- [ ] Footage duration sufficient for planned pacing
- [ ] B-roll available for coverage (if needed)
- [ ] Screen recordings clean (no distracting elements)
- [ ] Product UI in desired state (logged in, populated data, etc.)

#### Editorial Planning
- [ ] Video type identified (Demo/Hype/Tutorial/etc.)
- [ ] Target runtime defined (30s/60s/90s)
- [ ] Platform requirements known (aspect ratio, specs)
- [ ] Hook strategy selected (Curiosity/Problem/Time-Saving)
- [ ] CTA copy finalized
- [ ] Key proof moment identified (data bloom, visual payoff, etc.)

### 3.2 During-Edit Checklist

Run during active editing. Ensures quality and consistency.

#### Pacing
- [ ] Video type pacing rules followed (fast/medium/slow)
- [ ] No single scene longer than 15s (unless intentional hold)
- [ ] Scene changes feel motivated (not random)
- [ ] Music and cuts aligned (if beat-synced video)
- [ ] Hook lands within first 3-5s
- [ ] CTA appears in final 3-5s

#### Audio
- [ ] Music volume appropriate (-18 to -24 LUFS if no voiceover)
- [ ] Voiceover (if present) clear and audible above music
- [ ] No audio clipping or distortion
- [ ] Transitions don't cause audio pops/clicks
- [ ] Beat-synced cuts aligned to transients (if applicable)

#### Visual Hierarchy
- [ ] One focal point per shot (viewer knows where to look)
- [ ] Text readable at mobile size (min 18px effective size)
- [ ] High contrast between text and background
- [ ] Feature highlights direct attention appropriately
- [ ] Negative space used for clarity (not clutter)
- [ ] Light/dark alternation present (if 4+ scenes)

#### Technique Application
- [ ] Techniques match video type (no tutorial glitch cuts)
- [ ] Value-5 techniques prioritized for this type
- [ ] Transitions serve purpose (not decorative)
- [ ] Max 4 zoom punches per video (if any)
- [ ] Morph transitions connect logically
- [ ] Kinetic type legible and timed well

#### Brand Consistency
- [ ] Brand colors used appropriately
- [ ] Fonts match brand guidelines
- [ ] Logo appears if required
- [ ] Tone matches brand voice

### 3.3 Post-Edit QA Checklist

Run after completing the edit, before delivery. Final quality gate.

#### Technical Specs
- [ ] Resolution matches delivery spec (1080p, 4K, etc.)
- [ ] Frame rate correct (30fps, 60fps, etc.)
- [ ] Aspect ratio correct (16:9, 9:16, 1:1, etc.)
- [ ] Codec and bitrate meet platform requirements
- [ ] File size within platform limits
- [ ] Audio format correct (AAC, 48kHz typical)

#### Content Review
- [ ] Hook compelling and clear
- [ ] Core message communicated
- [ ] Visual proof element present
- [ ] CTA clear and actionable
- [ ] No spelling/grammar errors in text
- [ ] No placeholder content left in
- [ ] No unintended flashes or errors

#### Platform-Specific
- [ ] Safe zones respected (no text in outer 10%)
- [ ] Subtitles/captions if required
- [ ] Thumbnail frame selected (if needed)
- [ ] Video works without sound (if platform mutes by default)
- [ ] Branding visible in first frame (for autoplay)

#### Final Watch
- [ ] Full playthrough without skipping
- [ ] Watch on mobile if mobile-first platform
- [ ] Check on mute (text/visuals carry message?)
- [ ] Get second opinion if possible
- [ ] Export settings verified

## 4. Decision Trees

Structured decision-making for common editorial choices.

### 4.1 Hook Selection Decision Tree

**Goal**: Choose the right hook based on video type and audience intent.

```
START: What is the primary video type?
│
├─ Demo
│  ├─ Is the problem well-known? → YES → Problem-Based Hook
│  └─ Is the product novel/unexpected? → YES → Curiosity Gap Hook
│
├─ Hype
│  ├─ Is there a dramatic metric? → YES → Data Bloom Hook (start with number)
│  └─ Is there a philosophical angle? → YES → Manifesto Hook (bold statement)
│
├─ Tutorial
│  └─ ALWAYS → Problem-Based Hook (show the pain point)
│
├─ Compare
│  ├─ Is time a key benefit? → YES → Time-Saving Hook
│  └─ Is quality the differentiator? → YES → Problem-Based Hook (show old state pain)
│
├─ Testimonial
│  └─ ALWAYS → Problem-Based Hook (person's original problem)
│
├─ Brand
│  └─ ALWAYS → Manifesto Hook (philosophical statement)
│
├─ Explainer
│  └─ ALWAYS → Curiosity Gap Hook (question or surprising fact)
│
└─ Recap
   └─ ALWAYS → Data Bloom Hook (biggest achievement number)
```

**Hook Duration Guide**:
- Curiosity Gap: 3-5s
- Problem-Based: 3-8s (depends on complexity)
- Time-Saving: 2-4s
- Data Bloom: 3-5s
- Manifesto: 4-8s

### 4.2 Pacing Decision Tree

**Goal**: Determine scene length and cut frequency.

```
START: What is the video type?
│
├─ Hype / Recap
│  └─ Scene Length: 1-3s per scene
│      Cut Frequency: Every beat (60-80 cuts per minute if 120-140 BPM)
│      Transition Style: Beat-synced cuts, zoom punches
│
├─ Demo
│  ├─ Is this a feature overview? → YES
│  │  └─ Scene Length: 4-6s per feature
│  │      Cut Frequency: 8-12 cuts per minute
│  │      Transition Style: Morph transitions between features
│  │
│  └─ Is this a live interaction? → YES
│     └─ Scene Length: 6-10s per interaction
│         Cut Frequency: 6-10 cuts per minute
│         Transition Style: Minimal cuts, let typing/interaction play
│
├─ Tutorial
│  └─ Scene Length: 8-15s per step
│      Cut Frequency: 4-8 cuts per minute
│      Transition Style: Morph or geometric wipe between steps
│
├─ Compare
│  └─ Scene Length: 5-10s per comparison point
│      Cut Frequency: 6-12 cuts per minute
│      Transition Style: Split view holds, morph at state change
│
├─ Testimonial
│  └─ Scene Length: 6-12s per quote or proof point
│      Cut Frequency: 5-10 cuts per minute
│      Transition Style: Fades or gentle wipes
│
├─ Brand
│  └─ Scene Length: 8-15s per concept
│      Cut Frequency: 4-8 cuts per minute (beat-synced to musical phrases)
│      Transition Style: Fades, atmospheric transitions
│
└─ Explainer
   └─ Scene Length: 6-10s per concept
       Cut Frequency: 6-10 cuts per minute
       Transition Style: Morph transitions showing concept evolution
```

### 4.4 Technique Selection Decision Tree

**Goal**: Choose techniques based on editorial goal.

```
START: What are you trying to communicate in this scene?
│
├─ Product Reveal (first appearance)
│  ├─ Is it a UI/screen? → YES → 3D Tilt Entry + Foreground Orb
│  ├─ Is it a physical product? → YES → Environmental Container (device frame)
│  └─ Is it multiple products? → YES → Cascading Windows
│
├─ Feature Demonstration
│  ├─ Is it interactive (typing/clicking)? → YES → Live Typing + Feature Highlight
│  ├─ Is it a workflow (multi-step)? → YES → Morph Transition between steps
│  └─ Is it a UI detail? → YES → Feature Highlight (callout)
│
├─ Comparison (before/after)
│  ├─ Side-by-side needed? → YES → Split View
│  ├─ Same UI, different data? → YES → Full-Circle Visual Payoff
│  └─ Transformation? → YES → Morph Transition
│
├─ Social Proof (testimonial/metrics)
│  ├─ Is it a number? → YES → Data Bloom
│  ├─ Is it a quote? → YES → Decorative Symmetry + Negative Space
│  └─ Is it a person? → YES → Lower Third
│
├─ Excitement/Energy
│  ├─ Is music beat-driven? → YES → Beat-Synced Cut
│  ├─ Is this a high-impact moment? → YES → Zoom Punch
│  └─ Is this text-focused? → YES → Kinetic Type
│
├─ Clarity/Education
│  ├─ Is it a step-by-step? → YES → Negative Space + Feature Highlight
│  ├─ Is it a concept? → YES → Morph Transition (concept evolution)
│  └─ Is it a process? → YES → Geometric Wipe between stages
│
├─ Brand/Emotion
│  ├─ Is it philosophical? → YES → Kinetic Type + Dark Background
│  ├─ Is it atmospheric? → YES → Blur + Parallax + Ambient Particles
│  └─ Is it a value statement? → YES → Manifesto-Demo-Manifesto (A-B-A)
│
└─ Breadth (showing many things)
   ├─ Multiple features? → YES → Horizontal Scroll Gallery or 3D Text Cloud
   ├─ Multiple achievements? → YES → Cascading Windows
   └─ Multiple products? → YES → Panoramic Blur (bg montage)
```

## 5. How This Playbook Is Used During Editing

This playbook serves as the systematic knowledge base for the video editing agent. Here's how it integrates into the production workflow.

### 5.1 Input Processing

When the agent receives an editing task, it:

1. **Identifies Video Type** using taxonomy in Section 1
2. **Loads Template Checklist** for that type
3. **Selects Hook Strategy** using Decision Tree 4.1

### 5.2 Scene-by-Scene Editing

3. **Validates Against During-Edit Checklist** (3.2)
4. **Verifies Pacing** matches video type profile
5. **Ensures Visual Hierarchy** (focal point, text readability, negative space)

### 5.3 Quality Gates
